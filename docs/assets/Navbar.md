# UX/UI settings — Navbar (Aeria)

Краткий UX-контракт для navbar. Детальные системные правила — в `docs/Guide.md`.

## 1) Роль navbar
- Единый системный слой навигации на desktop и mobile.
- Не ломает контекст чтения и не меняет высоту скачкообразно.

## 2) Режимы
1. Основная навигация: `Главная`, `Эпизоды`, `Персонажи`, `Атлас`.
2. Detail-режим: `Назад` + breadcrumbs.
3. Reading-режим: счётчик `current / total` + progress-strip под navbar.

Detail breadcrumbs:
- реализованы через системный `components/ui/breadcrumb` (shadcn-паттерн);
- separator фиксирован как `>`, spacing управляется нашими токенами.

## 3) Быстрые действия
- `Поиск` (inline-панель под navbar, горячая клавиша `Ctrl/Cmd + K`).
- `День/Ночь`.
- `Aa` (UI/Headings/Text).
- `...` (Theme/Tap effect).
- Все dropdown-поля внутри `Aa` и `...` собираются через системный `Select`, а не через browser-native `select`.
- У `Select` в этих панелях обязательна заметная hover/focus/open реакция у trigger и option-items.
- В `Style`-селекте темы выводятся с цветными акцентными точками рядом с названиями.

Все настройки сохраняются в cookie `aeria-theme`.

## 4) Surface и ритм
- Верхний navbar: `sticky`, контент в `width-wide`.
- Внутренние inset desktop-navbar: сверху `--space-lg`, снизу `--space-md`.
- Mobile: нижняя панель с теми же разделами.
- В desktop правой группе:
  - по умолчанию `Поиск`, разделитель, `День/Ночь`, `Aa`, `...`;
  - на чтении `Поиск`, разделитель, `current / total`, разделитель, `День/Ночь`, `Aa`, `...`.
- Панели `Aa` и `...` открываются в фиксированной правой зоне под navbar.

Глобальный якорь `Наверх`:
- не входит в navbar, но выравнивается по той же правой границе `width-wide`;
- не подходит к краям экрана ближе `--space-xl`;
- на desktop сидит на нижнем inset `--space-lg`, а на mobile поднимается над нижним navbar с тем же `lg`-ритмом;
- использует `/assets/icons/aeria-arrow.svg`, который по умолчанию хранится направленным вправо и поворачивается в коде по месту;
- визуально это только знак без отдельного outline-кольца и без заливки.

## 5) Глобальный поиск
- Ищет по эпизодам, персонажам, атласу, сериям, странам/локациям.
- SPA-переходы без перезагрузки.
- Пустой запрос показывает recent; при отсутствии recent заглушка не выводится.
- Состояния текста:
  - placeholder: `Поиск по главам, персонажам и миру...`
  - empty: `Ничего не найдено. Попробуйте другое слово.`
  - error: `Поиск временно недоступен. Попробуйте позже.`
  - loading: `Ищем...`

## 6) Единая underline-система (обязательно)
Локальные копии underline запрещены.

Допустимые классы:
- `ui-underline`
- `ui-underline-click`
- `ui-underline-hover`

Активация click underline:
- `.ui-underline-active`
- `data-underline-active="true"`
- `aria-current="page"`
- `aria-expanded="true"`

К этим правилам обязаны соответствовать:
- разделы navbar,
- кнопки `Поиск`, `Aa`, `...`,
- кнопки фильтров в каталогах персонажей и эпизодов.
